# 強制更新所有 Portal 到 GitHub

## 問題
GitHub 上還是舊資料，需要強制更新

## 解決方案

### 方法 1: 使用完整部署腳本（推薦）

雙擊執行：
```
完整部署.bat
```

### 方法 2: 手動執行命令

在 PowerShell 或命令提示符中執行：

```bash
cd "c:\Users\keith\OneDrive\Desktop\Profile"

# 1. 檢查狀態
git status

# 2. 強制添加所有文件
git add -A
git add --all

# 3. 提交（使用時間戳）
git commit -am "Force update all portals - $(Get-Date -Format 'yyyy-MM-dd HH:mm:ss')"

# 4. 強制推送到 GitHub
git push --force origin main

# 如果 main 分支失敗，嘗試 master
git push --force origin master
```

### 方法 3: 如果還是不行，嘗試完全重置

```bash
cd "c:\Users\keith\OneDrive\Desktop\Profile"

# 檢查當前分支
git branch

# 添加所有文件
git add -A

# 提交
git commit -m "Complete update all portals"

# 強制推送（會覆蓋遠程）
git push --force origin main
```

推送完成後：
1. 訪問 GitHub 倉庫
2. 檢查文件修改時間
3. 查看最近的 commit 記錄
4. 確認所有 portal 文件都已更新

## 如果問題仍然存在

請執行以下檢查：

```bash
# 檢查遠程倉庫 URL
git remote -v

# 檢查分支
git branch -a

# 檢查本地和遠程的差異
git log origin/main..HEAD

# 查看所有未推送的提交
git log --branches --not --remotes
```
