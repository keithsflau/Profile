第一課資訊處理簡介 (Intro to Info Processing)

重點精華整理

數據 (Data) 與 資訊 (Information) 辨析

  • 數據 (Data)：一組未經整理的事實，單獨來看通常沒有任何意義（例如：未經處理的氣溫數字、未剪輯影片）。
  • 資訊 (Information)：數據經處理（排序、計算、分析等）後，轉換為有意義的輸出（例如：公司的年淨收入增長率、逐字演講稿）。

輸入-處理-輸出週期 (IPO Cycle)

資訊系統的基本運作架構。例如將食材輸入烤箱，烤箱進行烘烤（處理），最終輸出香噴噴的披薩。

資訊處理的七個階段

1. 收集 (Collection)：觀察、實驗、問卷調查。
2. 組織 (Organization)：分類與有效性檢驗。
3. 儲存 (Storage)：存於硬碟、快閃記憶體。
4. 處理 (Processing)：編輯、修改、多媒體更新。
5. 分析 (Analysis)：提取趨勢，如分析氣候。
6. 傳輸 (Transmission)：利用網絡媒體傳送數據。
7. 演示 (Presentation)：通過文字、圖像、音效等媒體輸出資訊。

第二課數據控制及組織 (Data Control & Org)

重點精華整理

「無用輸入，無用輸出」(GIGO - Garbage In, Garbage Out)

輸入錯誤數據，只會導致無法得出有用結果。數據輸入常見錯誤有：
1. 數據來源錯誤 (Source Error)：用戶提供了不存在的值（如錯誤的電子郵件）。
2. 轉錄錯誤 (Transcription Error)：數據誤植（如將「冶」打錯為「治」）。
3. 換位錯誤 (Transposition Error)：相鄰數字位置調轉（如「$15」變成「$51」）。

數據有效性檢驗 (Data Validation) 一覽

  • 完整性檢查 (Completeness Check)：確保必填欄不為空白。
  • 固定值檢查 (Fixed Value Check)：只允許特定範圍的值（如單選鈕 M/F）。
  • 類型檢查 (Type Check)：確保數據為特定類型（如年齡必須是數字）。
  • 範圍檢查 (Range Check)：數值位於指定區間內（如年齡必大於12）。
  • 長度檢查 (Length Check)：字元數量符合要求（如8位電話號碼）。
  • 格式檢查 (Format Check)：符合特定格式（如電郵地址須包含 @ 符號）。

數據驗證 (Data Verification)

確保數據在傳輸或轉移過程中準確一致。常見方法有：
• 輸入數據兩次作驗證 (Enter Data Twice)：由同一個操作員重複輸入兩次密碼。
• 雙重數據輸入 (Double Data Entry)：由兩個操作員分別輸入同一個檔案，再比對兩者。
• 校對數據 (Visual Check / Proofreading)：將螢幕輸入內容手動比對紙本原稿。

奇偶檢測 (Parity Check) 算法

在二進制代碼末尾，添加一個「奇偶檢驗位 (Parity Bit)」：
• 偶數檢測 (Even Parity)：使整個傳輸代碼中 $1$ 的總數為偶數。
• 奇數檢測 (Odd Parity)：使整個傳輸代碼中 $1$ 的總數為單數。

第三課數據表示 (Data Representation)

重點精華整理 (必考核心)

二進制補碼 (Two's Complement)

用於在 $N$ 位元中表示有符號整數 (Signed Integers)：
• 正數與無符號整數完全一樣，首位元為 $0$。
• $N$ 位元二進制補碼可表示的範圍：$-2^{N-1}$ 至 $2^{N-1}-1$。
• 求 $-X$ 的二進制補碼的快捷步驟：
  1. 將正數 $X$ 轉換成二進制，並在前補 $0$ 至 $N$ 位。
  2. 反轉所有數位 (Invert bits)：$0$ 變 $1$、$1$ 變 $0$。
  3. 加 1 (+1) 即得補碼。
例如：4-bit 系統中表示 $-3_{10}$：
$3_{10} = 0011_2 \xrightarrow{\text{反轉}} 1100_2 \xrightarrow{+1} 1101_2$

溢出誤差 (Overflow Error) 原理

發生在計算結果超出了系統所能表示的極限數值範圍。
• 有符號相加：若兩個正數相加得到負數，或兩個負數相加得到正數，必定發生溢出。
• 注意：對於有符號數，多出一個進位位元（Carry-out）並不一定代表發生了溢出！必須判斷最終結果是否在範圍內。

多媒體容量計算公式

  • 未經壓縮圖像 (Bitmap) 大小：
    $$\text{解像度 (寬} \times \text{高)} \times \text{色深 (bit)} = \text{大小 (bit)}$$
  • 未經壓縮音訊 (WAV) 大小：
    $$\text{取樣頻率 (Hz)} \times \text{位元深度 (bit)} \times \text{聲道數 (Channels)} \times \text{長度 (秒)} = \text{大小 (bit)}$$
  • 未經壓縮視訊 大小：
    $$\text{單幀大小 (bit)} \times \text{幀速率 (fps)} \times \text{長度 (秒)} + \text{音軌大小}$$
  • 注意：由位元（bit）轉換為字節（Byte）需除以 $8$，再轉換為 $\text{MB}$ 需連續除以 $1024$ 兩次（即除以 $1024^2$）。
第四課試算表 (Spreadsheets)

重點精華整理

相對引用與絕對引用 (Cell References)

在拖曳、複製、自動填滿公式時：
• 相對引用 (e.g. A1)：其參照的儲存格隨著複製位置相對移動。
• 絕對引用 (e.g. $A$1)：行和列都被 $ 號「鎖定」，不論公式複製到哪裡，參照的儲存格永遠固定。
• 混合引用 (e.g. $A1, A$1)：只鎖定欄名（複製時行變列不變）或只鎖定列號（複製時列變行不變）。

試算表必考預設函數

  • SUMIF(range, criteria, [sum_range])：依條件加總。例如 `SUMIF(C:C, ">=50", E:E)`。
  • VLOOKUP(value, table, col, [approx])：垂直查閱。第二個引數的對照表範圍通常需要用絕對引用（如 `$I$2:$J$5`），否則向下拉公式時對照表會錯位！
  • INT(number) vs ROUNDDOWN(number, 0)：`INT(-2.5) = -3`（向下至最接近整數），而 `ROUNDDOWN(-2.5, 0) = -2`（向 0 方向捨入）。

數據分析工具：樞紐分析表與目標搜尋

1. 樞紐分析表 (Pivot Table)：最適合用來匯總、分類、統計、比較大型和複雜的數據集（如分析不同書籍類別的銷售總額），而不需要手動編寫多重 `SUMIF` 或 `COUNTIF`。
2. 目標搜尋 (Goal Seek)：在已知理想輸出目標的情況下，自動逆向計算出一個變數所需的輸入值。如：「如果我想總利潤達到 $100000$（目標），我需要賣出多少本書（變數）？」。

第五課數據庫 (Database & SQL)

重點精華整理

關係數據庫基本概念

• 主關鍵碼 (Primary Key)：用以唯一標識每條記錄的欄位。主關鍵碼不能重複，也不能為空值 (NULL)。由多個欄位組成的主關鍵碼稱為「複合鍵 (Composite Key)」。
• 輸入遮罩 (Input Mask)：限制並強制要求數據必須以特定格式輸入。如 `0000\-0000` 表示必須輸入八位數字（例如電話號碼）。

Access 的四大物件物件

資料表 (Table)
數據庫核心，以列（記錄）和行（欄位）儲存數據。
表單 (Form)
提供直觀的人機界面，每次只顯示並編輯一條記錄。
查詢 (Query)
利用 SQL 語句提取、篩選、計算、排序或刪除數據。
報表 (Report)
自訂排版並進行分組與匯總，專門用作輸出或列印。

SQL 核心語法與關鍵點

  • DISTINCT：篩選掉重複值。例如 `SELECT DISTINCT CLASS FROM STUDENT`。
  • Access 中的萬用字符：
    • `*` (代表任意數量的字元)
    • `?` (代表只有一個字元)
    例如 `WHERE CNAME LIKE '*方*'` 或 `WHERE CNAME LIKE '??'`。
  • 分組篩選：
    • WHERE 子句：在分組 (GROUP BY) 前對每條記錄進行篩選。
    • HAVING 子句：在分組 (GROUP BY) 後，對組別進行條件篩選。
  • 空值檢索：必須使用 `IS NULL` / `IS NOT NULL`，千萬不能用 `= NULL`！
